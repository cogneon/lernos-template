2基本

2.1lernOSガイドの基本的な考え方

lernOSガイドは、学習者が学習スプリント（3ヶ月）の枠組みの中で、事前に定義された学習パスを通して自分の学習目標を達成することを支援します。学習者は、一人で、あるいは学習タンデムで、あるいは学習サークル（4～5人）で学びます。スプリントに必要な時間は、1週間に2時間を超えないようにしてください。学習目標は、これらの分野の1つまたは複数の組み合わせから得ることができます。

  • MINDSET（マインドセット）：あるべき姿勢を身につけること
  • SKILLSET- スキルの習得
  • TOOLSET- ツールやメソッドを使いこなす

ラーニングスプリントの第0週と第12週は常に計画と回顧のためのもので、第4週と第8週には学習の進捗状況を中間的に振り返るためのいわゆるピットストップがあります。第1週から第11週までは、カタと呼ばれるエクササイズ（最低11回）で埋め尽くされます。1つの型は、練習者が30～60分かけて、A4用紙2枚以内（寿司カードの原則）に記載する必要があります。

lernOSガイドは常にCreative Commons Attribution 4.0 International License(CC BY)でライセンスされています。このライセンスは、ガイドに使用されている画像にも適用されます。また、すべてのコンテンツは、編集可能なソースフォーマット（画像の場合、PNGやSVGなど）で提供されなければなりません。lernOSガイドの内容は、GitHub上のリポジトリとして管理され、オープンアクセスと再利用のしやすさを確保しています。

これにより、LernOSのコンテンツは、あらゆる目的（商用利用を含む）で、オープンにアクセス、使用、変更、共有することができます。ただし、製品やサービスの名称に「lernOS」を冠してはならない。このアプローチは、Chromiumブラウザと、それをベースにした製品であるGoogle Chrome、Microsoft Edge、Opera、Braveに匹敵するものです。

2.2ガイドラインのライフサイクル

アイデアから完成まで、LernOSガイドは様々な段階を経ますが、これをガイドのライフサイクルと呼んでいます。それぞれのガイドの歴史は違っていても、大まかには次のようなフェーズが挙げられる。

  • アイデア- 個人または個人のグループが、LernOS Learning PathまたはLernOS Guideのアイデアを持っています。
  • 目標の定義- アイデアがより具体的になり、学習目標と学習者のターゲットグループが明確になります。多くの場合、最初のコンテンツは、スケッチやメモ、メールのやり取りなどの形で、すでにここで作られています。
  • 作成- この段階では、構造とコンテンツが作成されます。ラーニングパスはWord、Google Doc、PowerPointなどのフォーマットで作成できますが、lernOSガイドは常にMarkdownをソースフォーマットとしています。
  • 公開- ガイドはGitHubのリポジトリとして公開されます。Markdownに加えて、PDF、Word、HTML、電子書籍（EPUB、Mobipocket）の各フォーマットがデフォルトで作成されます。ガイドの学習経路がまだ実際にテストされていない場合は、0.xのバージョン番号を推奨します。このガイドが実際にテストされた後、バージョン番号1.0が与えられ、国際的な使用を可能にするために、少なくともドイツ語と英語で提供されなければなりません。
  • プロモーション- ガイドを宣伝し、学習者に使用を促す（フィードバックを集める上で重要）。これは、例えば、lernOSのコミュニティ・プラットフォームであるCONNECT、lernOS Community Call、Twitterアカウントを通じて行うことができます。 @lern_osツイッターアカウント
  • メンテナンス- この段階では、著者（メンテナー）が、理想的には共著者のチームと一緒に、ガイドの維持、コンテンツの拡大、フィードバックの取り込みを行います。一定期間ごとにガイドを更新し、継続的に改善していくことをお勧めします（例：四半期ごと、半年ごとなど）。さらなる開発計画を立てるには、GitHubのプロジェクトボードやTrelloのボードなど、ボーイズボードが適しています。
  • アーカイブ- ガイドがメンテナンスされなくなり、コンテンツが古くなって使えなくなった場合、コンテンツはアーカイブされますが、引き続き利用可能です。

参照してください。

2.4目次と構成

学習者が簡単にオリエンテーションできるように、すべてのlernOSガイドは同じ基本構造を持っています（ブログ）。これは、後述する「About lernOS」「Basics」「Learning Path」「Appendix」の4章で構成されています。

2.4.1「lernOSについて」の章

lernOSについて」の章では、lernOSの基礎知識を説明しています（すべてのガイドで共通です）。

2.4.2「基礎」の章

基本」の章では、ガイドラインのテーマを理解するために必要な理論的背景や情報を掲載しています。本章では、他の利用可能なソースや文献（ハブまたはキュレーション機能）についても多く言及されています。

2.4.3「ラーニングパス」の章

ラーニングパス」の章には、ガイドのカタ（練習）が書かれています。それぞれの型はそれぞれの章で説明されており、その長さは最大でも印刷された2ページ分でなければなりません（寿司カードの原則）。1つのガイドには、複数のラーニングパスが含まれる場合があります。

kataのドキュメントに関するWikiページ。

2.4.4章「付録

付録の章には、少なくとも「変更履歴」が含まれており、多くの場合、謝辞（貢献者の名前など）の章も含まれています。

2.5コンテンツ、メディア、ソース

2.5.1書き方

lernOSのガイドをできるだけ利用しやすいものにするために、言語はできるだけシンプルにする必要があります。英単語や専門用語は可能な限り避けてください。使われている専門用語は、常に一般的な言葉で説明しなければならない。専門用語は、すべてのlernOSガイドで一貫した用語の使用を保証するために、lernOS用語集で定義されている通りに使用しなければなりません。多くの専門用語が紹介されているガイドの場合、ガイドの付録の中にGlossaryのサブチャプターを設けることが望ましいです。

ラーニング・パス」の章では、学習者に直接「あなた」と声をかけることで、活性化を図っています。

2.5.2画像

百聞は一見に如かず。だからこそ、ガイドを描くことには大きな意味があります。また、画像についても、CC BYライセンスのソースを必ず使用してください。ベクターグラフィックスの場合はSVG（ガイドではPNGとして掲載）、ピクセルグラフィックスの場合はPNGが好ましいフォーマットです。SVGベクターグラフィックスの作成・編集やPNGへの書き出しにはInkscape、ピクセルグラフィックスにはGimpをお勧めします。

2.5.3オーディオとビデオ

プロダクションチェーンではテキスト文書も生成されるため、オーディオやビデオのコンテンツをlernOSガイドに埋め込むことはできません。とはいえ、音声や動画のコンテンツを参照するには、文章中にリンクを貼ればよい。

  • Microsoft Word- Office 365の一環としてのオンライン翻訳サービス（レビュー/翻訳/文書の翻訳）

2.6プロダクションチェーンガイド

lernOSガイドはMarkdownマークアップ言語で作成され、通常GitHubで管理されます。PDF、Word、e-bookなどのさらなるターゲットフォーマットは、lernOSプロダクションチェーンを介してMarkdownソースから生成されます。この生成は、GitHub（まだ利用できません）で集中的に行うことも、自分のコンピュータで行うこともできます。対象となるフォーマットを自分のコンピュータで生成する場合には、以下に述べるようなツールセットが必要となります。可能な限りオープンソースのソフトウェアを使用しています。

2.6.2Windows 10でのlernOSプロダクションチェーン

いわゆるMakefile（make.bat）は、マークダウンソースからターゲットフォーマットを作成するために使用されます。自分のコンピュータでプロダクションチェーンが正しく設定されていれば、Makefile（make.bat）をダブルクリックすると、すべてのターゲットフォーマットが自動的に生成される。お使いのコンピューターのサイズや速度によっては、数分かかることがあります。プロダクションチェーンでは、次のようなステップが行われます。

  1. 制作したドキュメントの古いバージョンを削除する。
  2. ワード版（.docx）の作成
  3. HTML版（.html）の作成（CMSやESNへのコピー＆ペースト用の生のHTMLなど）
  4. ウェブ版の制作（テーマ素材を使ったmkdocs、結果はdocsフォルダへ）
  5. PDF版（.pdf）の作成
  6. 表紙をPDFからグラフィックとして書き出す (src/images/ebook-cover.jpg)
  7. 電子書籍版EPUB（.epub）の制作
  8. 電子書籍のEPUB版（.epub）をMobipocket版（.mobi）に変換する。
  9. 準備完了 :-)

2.6.3lernOSのプロダクションチェーンを拡張するためのアイデア

ガイドと同様に、lernOSのプロダクションチェーンも継続的に拡大しています。拡大のためのいくつかのアイデア。

  • Github Actions：現在、GitHub Actionsによる自動化されたプロダクションチェーンを準備中です。
  • OneNote：OneNote版のガイドを追加作成しました（現在は添付文書としてのみ提供しており、手動で管理する必要があります）。
  • オーディオブックです。チャプターマーカーをつけて聴けるオーディオブックの制作（例：Balbolkaを使ったもの）。

2.7プロダクションチェーン用ツールセット

lernOSのプロダクションチェーンには、自分のコンピュータやクラウド上にあるいくつかのツールが必要です。

  1. Markdownエディタ- ガイドの内容を編集するためのもので、すべてMarkdown（Typora、Atom、Mark Textなど）で書かれています。

  2. Github Desktop- GitHub上のガイドのリポジトリをローカルコンピュータと同期させることができます。

  3. Pandoc- MarkdownをlernOSプロダクションチェーンのターゲットフォーマットに変換します。

  4. LaTeX- PandocがPDFファイルを生成するために必要です。

  5. MkDocsとMaterial for MkDocs- Web版のガイドを生成するためのものです。

  6. ImageMagick- 電子書籍版に挿入するガイドのカバーページのグラフィックを作成する。

  7. Calibre- Pandocで作成したepub形式の電子書籍版をmobi形式に変換する。

2.7.1Typora

Typoraは、Markdownコンテンツを編集するためのテキストエディタです。Typoraは、プラットフォームを問わず利用でき、Markdownを真のWYSIWYG（What You See Is What You Get）として表示するため、lernOSコンテンツのMarkdownエディタとして推奨されています。Markdownは基本的にはシンプルなテキストエディタでも作成・編集が可能ですが、WYSIWYG表示は特に初心者には有利です。

2.7.2Pandoc

Pandocは、Markdownを他のフォーマットに変換できるコマンドラインベースのテキストコンバータです。MarkdownをMicrosoft Wordに変換した例。

pandoc document.md -o document.docx

Pandocがサポートしているフォーマットは以下の通りです。

  • アスキードック

2.7.3GitHubデスクトップ

Github Desktopは、GitHubのリポジトリをローカルコンピュータと同期させることができるクライアントです。これは例えば、Markdownコンテンツをローカルで作業し、必要に応じてリポジトリと同期させることができることを意味します。この機能は、DropboxやOneDriveとのファイルの同期に似ています。

2.7.4LaTeX & Co.

LaTeXは、TeX形式で書かれたテキスト文書を、HTMLやPDFなどの形式に変換するためのプログラムパッケージです。lernOSのプロダクションチェーンでは、LaTeXのディストリビューション（WindowsではMiKTeX、MacではMacTeX）を使って、pandocで直接PDFファイルを生成しています。そのためにはGhostscriptも必要となります。

pandocのLaTexテンプレートEisvogelは、lernOSガイドのPDF作成用テンプレートとして使用され、metadata.yamlファイルによって適応されます。このテンプレートを初めて使用する際には，さらにLaTeXの拡張機能がインストールされます。

2.7.5MkDocs & Material for MkDocs

MkDocsは、ナビゲーション、検索、ソーシャルリンク、コメント機能を備えたレスポンシブウェブ版を作成する静的ウェブサイトジェネレーターです。MkDocsの素材は、lernOSガイドのウェブ版のテーマとして使用されています。

Web版の各ページの最後にあるコメント機能には、Disqusサービスを使用しています。

2.7.6キャリバー

Calibreは電子書籍管理ソフトで、電子書籍版のEPUBをMobipocket（mobi、Kindle ReaderやKindle App用など）に変換する際に使用します。PDF版のガイドから表紙の画像をグラフィックとして抽出し、電子書籍に適したサイズに拡大するために、グラフィックパッケージ「ImageMagick」を使用しています。

3ラーニングパス

ラーニングパスの11のカタの最初のスケッチ。

ピットストップ1まで - テクニカルインフラ

  • 第1週 - Markdownを知ろう：Markdownエディタ（推奨Typora）をダウンロードして、Markdown構文の最初の一歩を踏み出しましょう。Typoraの利点：プロダクションチェーン全体を必要とせずに、すでにPDF、Word、HTMLなどを書き出すことができます（初期の達成感）。
  • 2週目 - GitHubからテンプレートをダウンロード： lernos-templateリポジトリから、ガイドテンプレートをZIPファイルでダウンロードし（右上：Code -> Download ZIP）、ローカルで解凍します。ディレクトリとファイルの構造を理解してください。
  • 第3週 - プロダクションチェーンの設置プロダクションチェーン用のツールセットをインストールし、プロダクションチェーンを稼働させます。テンプレートや制作ガイドのメタデータやコンテンツに変更を加える。
  • 第4週 - GitHubの同期を設定する。GitHubアカウントの作成、GitHub Desktopのインストール、ローカルのテンプレートディレクトリとGitHubリポジトリの同期。コミットとmaster/developブランチの違いを理解する（Gitflowも参照）。

Until Pit Stop 2 - Structure コンテンツとソース。

  • 第5週 - 目標設定：ガイドがどの学習者（ターゲットグループ）に対して、どの学習目標（マインドセット、スキルセット、ツールセット）を満たすべきか？ターゲットグループの代表者からフィードバックを得る。
  • 第6週 ブレインストーミング 目次4つの標準的な章（About lernOS、Basics、Learning Path、Appendix）に沿って目次をどのように構成するか、ブレインストーミングを行います。これは、例えば、マインドマップ（XMindなど）の形で行うことができ、構造を非常に柔軟に扱うことができます。
  • 第7週 - 3階層の目次。3つ（最大4つ）のレベルでチャプター構造を定義します。各章ごとに、内容の最初のこだわりポイントがすでにここに記録されています。
  • 第8週 - ソースのキュレーション基礎編で参照すべき資料を特定する。特に注意すべき点は、理論や実践におけるソースの確保（誇大広告や流行などの禁止）、長期的な利用可能性（機関の支援など）、オープンアクセスなどです。

レトロスペクティブまで - コンテンツ

  • 第9週 - ブックスプリント 1～5日間のブックスプリントでは、ガイドのコアコンテンツを作成します。目標は、ブックスプリントの終わりに、バージョン0.1がGitHub上のすべてのターゲットフォーマットのリリースとして利用可能であり、ターゲットグループがフィードバックを与えることができることです（例えば、GitHubのissueとして、またはコメント可能なWord/Google Docバージョンを通して）。
  • 第10週：フィードバックと編集：ターゲットグループからのフィードバックをコンテンツに反映。文章の校正や微妙なニュアンスの改善などが行われ、バージョン0.2となりました。
  • 第11週 - Go Live!: バージョン0.2を別のリリースの形でGitHubに公開し、適切なチャネルを通じてターゲットグループに伝えます。理想的には、次のスプリントでラーニングサークルの何人かがガイドを試し、フィードバックを得て、それに基づいてバージョン1.0を発行することができます。
